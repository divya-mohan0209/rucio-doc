Details

POST (/rules/()

Create a new replication rule.

  • Request JSON Object
* **dids** (*list*) – List of data identifiers.


* **account** (*string*) – Account issuing the rule.


* **copies** (*int*) – The number of replicas.


* **rse_expression** (*string*) – RSE expression which gets resolved into a list of RSEs.


* **grouping** (*string*) – ALL -  All files will be replicated to the same RSE.
DATASET - All files in the same dataset will be replicated to the same RSE.
NONE - Files will be completely spread over all allowed RSEs without any grouping considerations at all


* **weight** (*int*) – Weighting scheme to be used.


* **lifetime** (*int*) – The lifetime of the replication rule in seconds.


* **locked** (*string*) – If the is locked.


* **subscription_id** (*string*) – The subscription_id, if the rule is created by a subscription.


* **source_replica_expression** (*string*) – Only use replicas as source from these RSEs.


* **activity** (*string*) – Activity to be passed to the conveyor.


* **notify** (*string*) – Notification setting of the rule (‘Y’, ‘N’, ‘C’; None = ‘N’).


* **purge_replicas** (*bool*) – Purge setting if a replica should be directly deleted after the rule is deleted.


* **ignore_availability** (*bool*) – Option to ignore the availability of RSEs.


* **comments** (*string*) – Comment about the rule.


* **ask_approval** (*bool*) – Ask for approval for this rule.


* **asynchronous** (*bool*) – Create replication rule asynchronously by the judge-injector.


* **priority** (*int*) – Priority of the rule and the transfers which should be submitted.


* **split_container** (*bool*) – Should a container rule be split into individual dataset rules.


* **meta** (*string*) – Dictionary with metadata from the WFMS.
  • Status Codes
* [201 Created](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.2.2) – rule created


* [401 Unauthorized](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.2) – Invalid Auth Token


* [404 Not Found](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.5) – DID not found


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Invalid Replication Rule


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Duplicate Replication Rule


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Insufficient Target RSEs


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Insufficient Account Limit


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Invalid RSE Expression


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Replication Rule Creation Temporary Failed


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Invalid Rule Weight


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Staging Area Rule Requires Lifetime


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Scratch Disk Lifetime Conflict


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Manual Rule Approval Blocked


* [409 Conflict](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.10) – Invalid Object

  • Returns

    List of ids for created rules

* [Content-Type](https://tools.ietf.org/html/rfc7231#section-3.1.1.5) – application/x-json-stream

* [200 OK](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.2.1) – Rule found


* [401 Unauthorized](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.2) – Invalid Auth Token


* [404 Not Found](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.5) – no rule found for id


* [406 Not Acceptable](http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.4.7) – Not Acceptable

* **scope** – The scope name.
